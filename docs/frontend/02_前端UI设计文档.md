## 一、设计理念与风格定义

### 1.1 核心设计方向：「现代文墨」

**设计哲学**：将中国传统水墨美学与现代简洁UI融合——不是复古堆砌，而是用现代设计语言重新诠释古典气质。整体感觉：**清雅、克制、有书卷气，但绝不土气**。

**关键词**：墨迹质感 · 毛笔线条感 · 留白美学 · 现代扁平 · 沉稳不沉闷

**设计差异化**：
- 主色调选用「墨蓝 + 暖米」，区别于市面大量红黄系传统教育产品
- 排版大量运用「留白」，营造书页阅读的呼吸感
- 字体选用刚健的现代宋体（标题）+ 清晰无衬线（正文），两者结合有文学氛围
- 背景用细腻纸纹或轻微噪点，模拟宣纸质感（CSS实现，不是图片）

## 二、视觉设计系统

### 2.1 色彩系统

#### 主调色板

```css
:root {
  /* === 主色系：墨蓝 === */
  --ink-900: #0D1B2A;   /* 最深墨色，用于重要文字 */
  --ink-700: #1B3358;   /* 主色，导航背景、主按钮 */
  --ink-500: #2E5F8A;   /* 中间调，次级按钮、链接 */
  --ink-300: #6B9EC7;   /* 浅蓝，图标、装饰线 */
  --ink-100: #D4E8F5;   /* 极浅蓝，hover背景 */
  --ink-50:  #EEF6FC;   /* 最浅，card背景 */

  /* === 辅色系：暖米 === */
  --warm-900: #5C3D1E;  /* 深暖棕，暖调文字 */
  --warm-700: #A0622B;  /* 朱砂橙，强调色/标注 */
  --warm-500: #D4863C;  /* 金橙，积极反馈、成就 */
  --warm-300: #E8B87A;  /* 浅金，hover暖色 */
  --warm-100: #F5E6D0;  /* 浅米，卡片暖底色 */
  --warm-50:  #FDF6ED;  /* 最浅米，页面背景 */

  /* === 功能色 === */
  --success:  #2E7D55;  /* 正确/成功，墨绿 */
  --success-bg: #EBF5F0;
  --error:    #C0392B;  /* 错误/警告，朱红 */
  --error-bg: #FDECEA;
  --neutral:  #6B7280;  /* 次级文字 */
  --border:   #E2E8F0;  /* 默认边框 */
  --divider:  #F0F2F5;  /* 分隔线 */

  /* === 背景系 === */
  --bg-page:    #FDFAF7;  /* 整体页面背景，极浅暖米 */
  --bg-surface: #FFFFFF;  /* 卡片/弹窗背景 */
  --bg-subtle:  #F7F4F0;  /* 次级区块背景 */
}
```

#### 色彩使用规则
- 墨蓝色系：导航、主按钮、重要标题、链接
- 暖米色系：辅助信息、标签、积极状态、成就
- 红色（朱砂橙）：错误提示、重要警告、错题标注
- 绿色（墨绿）：正确、通过、成功
- 大面积使用 `--bg-page` 暖白背景，不用纯白，减少刺眼感

### 2.2 字体系统

```css

:root {
  /* 标题字体：宋体，文学感 */

  /* 正文字体：黑体，易读 */
  --font-body: 'Noto Sans SC', 'PingFang SC', 'Microsoft YaHei', sans-serif;

  /* 诗词原文专用（更大字号，宋体） */
  --font-poem: 'Noto Serif SC', serif;

  /* 代码/数字（等宽） */
  --font-mono: 'JetBrains Mono', 'Courier New', monospace;
}
```

## 三、核心组件设计规范

### 3.1 导航栏（Navbar）

**规格**：
- 高度：64px
- 背景：`--ink-700`（墨蓝）
- 左侧：LOGO（毛笔风格「诗」字图标 + 「诗境通」文字，白色宋体）
- 中部：主导航链接（探索 / 学习 / 练习 / 考试 / 我的）
- 右侧：用户头像 + 积分/连续天数徽章

**Logo 设计**：
```
[毛笔笔触感"诗"字SVG图标]  诗境通
```
- 图标：深蓝背景，墨蓝描边，金色笔触装饰
- 字形：使用宋体，字距略宽

**导航链接样式**：
- 默认：白色，透明度70%
- 激活：白色100% + 底部金色线条（2px暖橙色）
- Hover：白色100%，背景轻微变亮

**移动端**：汉堡菜单，底部Tab栏（5项）

### 3.2 诗词卡片（PoemCard）

```

│  [朝代标签]    [题材标签]        │

### 3.3 AI 解析结果卡片

每个解析模块为独立的折叠卡片：
```
┌───────────────────────────────────┐

- AI流式输出：使用打字机动画（每字淡入）
- 已完成的卡片：边框变为淡绿色，表示"已解析"
- 每个卡片右上角有「查看更多」或相关联动按钮

## 五、竞赛演示流程设计

### 5.1 演示时长规划（建议3-5分钟）

```
演示流程（5分钟完整版）

[0:00-0:30] 开场：首页 + 产品定位说明
  → 首页Hero区，展示「AI赋能古诗词学习」的概念
  → 简述痛点：背不会、不理解、考不好

[0:30-1:30] 核心演示1：AI智慧解析
  → 搜索「静夜思」（或提前准备好）
  → 点击「AI解析」，展示流式输出打字机效果
  → 展示：注释/译文/意象/考点 各板块逐步出现
  → 旁白：「学生不再需要死查字典，AI一键给出完整解析」

[1:30-2:30] 核心演示2：诗人穿越对话
  → 点击「聊一聊」→ 选择「李白」角色
  → 展示预设对话（事先练习好，流畅演示）
  → 旁白：「苏格拉底式引导，学生通过对话真正理解诗意」

[2:30-3:30] 核心演示3：AI题目 + 弱点诊断
  → 点击「练一练」，展示AI生成题目（10题，快速完成3-4题）
  → 答题→即时反馈→错题自动收录
  → 跳转「弱点诊断」，展示雷达图
  → 旁白：「系统精准识别薄弱点，不再漫无目的刷题」

[3:30-4:30] 核心演示4：模拟考试 + 复习计划
  → 展示模拟考试界面（倒计时、题目格式）
  → 快速跳至成绩单，展示AI阅卷结果
  → 展示AI生成的30天复习计划
  → 旁白：「AI阅卷 + 个性化复习计划，学习有据可依」

[4:30-5:00] 亮点总结
  → 展示学习数据统计（已学诗词/连续天数/进步曲线）
  → 展示AI创作点评（加分项）
  → 强调：「从理解到创作，完整学习闭环」
```

### 5.2 演示数据预埋

为保证演示流畅，需提前准备：

| 演示场景 | 预埋内容 |
|---------|---------|
| AI解析 | 「静夜思」解析结果提前缓存（避免等待） |
| 诗人对话 | 准备5-8轮精彩对话脚本（引导AI走好的方向） |
| 弱点图 | 预置一个「有弱点」的账号，雷达图有明显低谷 |
| 成绩单 | 预置一次模考记录，显示进步趋势 |
| 复习计划 | 预置一个「距中考30天」的计划，内容丰富 |

### 5.3 演示账号设置

创建「演示账号」：
- 已学诗词：80首（体现积累量）
- 连续学习：14天（打卡有热度）
- 有典型弱点（手法辨析偏弱，方便展示诊断价值）
- 有错题本记录（10+条）
- 有一篇AI点评过的创作

### 7.1 AI加载状态
```
AI正在思考...
[墨蓝色点点动画：● ● ●  依次亮起]
正在分析《静夜思》的意象...
```
- 显示当前进度文案（不只是转圈）
- 配合轻微的页面过渡动画

### 7.2 空状态
- 错题本为空：「太棒了，目前没有错题！去练习巩固一下吧 →」
- 无搜索结果：「没有找到这首诗，试试换个关键词？或者直接粘贴原文解析」

### 7.3 错误状态
- AI调用失败：「AI正在赶路，请稍候再试 🔄」（不显示技术错误）
- 网络错误：顶部toast提示，自动重试
